DESCRIPTION :

29Tasks is a simple productivity web app that I built with one clear goal in mind — to make staying consistent easier without overcomplicating things.

Most productivity tools today try to do too much. They come with too many features, too many options, and honestly, that can feel overwhelming. So instead of building something complex, I focused on creating something that is clean, straightforward, and actually usable every day.

29Tasks combines three important things: managing tasks, tracking time, and understanding your performance. These are the basics of productivity, and when they work together, they create a system that actually helps you improve.

## What this project does

At its core, 29Tasks lets you create tasks and decide whether you completed them or not. That’s it. There’s no unnecessary tagging system or complicated structure. You simply add a task, work on it, and then mark it as done or not done.

This might sound too simple, but that’s the point.

Instead of trying to be perfect, the app encourages honesty. Some days you’ll complete everything, some days you won’t — and that’s okay. What matters is that you’re tracking it.

## Timer features

Along with tasks, I added a stopwatch and a countdown timer.

The stopwatch helps you see how much time you’re actually spending on something. A lot of times we *think* we’re working for hours, but in reality, we’re distracted. This makes it clear.

The countdown timer is useful when you want to focus for a fixed amount of time, like study sessions or work blocks. You just set it and start.

Both tools are simple and don’t distract you while working.

## Performance tracking

One feature I personally find useful is the performance section.

It shows how many tasks you completed versus how many you didn’t. This gives a rough idea of how consistent you are. It’s not meant to judge you — just to give you awareness.

Over time, this helps you notice patterns:

* Are you consistent every day?
* Do you start strong and lose focus later?
* Are you setting too many tasks?

These small insights actually make a big difference.

## Design approach

I kept the design very clean and minimal. No heavy colors, no clutter, no unnecessary animations.

Everything is placed in a way that feels natural:

* Tasks are easy to add and read
* Navigation is simple
* Important information is visible without searching

The app works smoothly on different screen sizes, so you can use it on your laptop or phone without issues.

## Why I built this

I didn’t build 29Tasks to compete with big productivity apps. I built it because I wanted something that I would actually use daily.

Something that:

* Doesn’t distract me
* Doesn’t require setup time
* Doesn’t make me feel guilty
* Just helps me stay on track

And that’s what this project tries to do.

## Tech used

This project is built using basic web technologies:

* HTML
* CSS
* JavaScript

No frameworks, no heavy libraries. Just simple code that works.

This also makes it easier for anyone to understand the project and even modify it if they want to.

## Who can use this

Anyone can use 29Tasks, but it’s especially helpful for:

* Students trying to stay consistent with studies
* Beginners learning time management
* Developers tracking daily progress
* Anyone who wants a simple routine

## Future ideas

There’s still a lot that can be improved. Some ideas I have are:

* Saving tasks using local storage
* Adding user profiles
* More detailed performance insights
* Custom task categories

But even without these, the current version does its job well.

## Final thoughts

29Tasks is not about being perfect. It’s about showing up every day and doing what you can.

It’s a small tool, but if used regularly, it can actually help you build discipline and understand your habits better.

That’s all I wanted this project to be.
